Do
  • When users need to have a quick glance of any important entity of the activity they're performing.
  • Use it to display and encapsulate a set of related information in a non-linear or non-matrix form.
  • Feel free to use big icons, images and big typography. A card is a unique peace of design.
Don't
  • A set of cards is not a table. Don't use it to display data that should be easly comparable in a matrix.
  • A static card should not be used when there's more information left to display. They're not links.
  • Don't use the helper classes as a default for the design. They're just a starting point. Each card information design should be thoughtfully craft. Each card for a specific entity should have an unique design.
  • Don't use a card as a dialog or alert box.
